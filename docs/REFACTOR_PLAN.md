# Refactor Plan — Phrasongsa Timetable

> Generated by `/investigate` on 2026-05-26. Each item links back to a finding ID in [`CODE_REVIEW.md`](./CODE_REVIEW.md). Phases are ordered by impact / effort, not by priority within a phase — within a phase, items can be parallelized.

Companion docs: [`ARCHITECTURE.md`](./ARCHITECTURE.md), [`FEATURE_MAP.md`](./FEATURE_MAP.md), [`CODE_REVIEW.md`](./CODE_REVIEW.md).

## How to use this plan

- Each item names the **finding** it addresses, a rough **size** estimate, and a one-line **verification** check.
- File a `bd` issue per item, link the finding ID in the description, claim before starting.
- Verification commands assume the project's standard scripts: `pnpm typecheck`, `pnpm lint`, `pnpm test`, `pnpm test:e2e`, `pnpm build`.

## Phase A — Quick wins

Small, low-risk changes that close real security gaps and obvious foot-guns. Aim to land all of Phase A in a single short cycle.

### A.1 Reconcile `schema.prisma` with the database — **largely DONE (2026-05-26)**

- **Addresses**: C-1 (and the wider `schema.prisma` ↔ migrations drift it was a symptom of).
- **Root cause confirmed**: `prisma/generated/` is **gitignored** (not committed). The schema was pruned without re-running `prisma generate`, so stale local clients masked the drift; the first clean generate breaks the build.
- **Done in this pass** (email-removal work, issue `atf`):
  - `EmailOutbox` model + `EmailOutboxKind`/`EmailOutboxStatus` enums **dropped** (not restored) — email service removed. New migration `20260526120000_remove_email_outbox` drops the table + enums.
  - `breaktime` enum **corrected** in `schema.prisma` to match `0_init`: `BREAK_JUNIOR, BREAK_SENIOR, BREAK_BOTH, NOT_BREAK, BREAK` (was wrongly pruned to `BREAK, NOT_BREAK`). This was the same drift class — exposed only on a clean regenerate.
  - `prisma generate` re-run; `pnpm typecheck` ✅, `pnpm test` ✅ (768 passing).
- **Residual (still TODO)**: the better-auth `Verification` model is still absent from `schema.prisma` while a `verification` table exists in the DB (from `20251126125836_migrate_to_better_auth`). Harmless today — all flows that used it (password reset, email verification, email change) were removed with the email service, and typecheck/tests are green. Add the `Verification` model back to `schema.prisma` if any token-based better-auth flow is reintroduced. Audit other models for the same prune-without-regenerate drift (compare `schema.prisma` against `prisma/migrations/0_init`).

## Phase C — Long-term / strategic

Initiatives that should be planned with stakeholder agreement; not a one-PR fix.

### C.1 Tighten CSP for production

- **Addresses**: M-1.
- **Steps**:
  1. Split the `headers()` callback in `next.config.mjs` to emit different CSP values for `process.env.NODE_ENV === "production"` vs dev.
  2. Drop `'unsafe-eval'` from production CSP; verify MUI Emotion SSR + React Compiler still work.
  3. If they don't, file an upstream issue / find a workaround; meanwhile, keep `'unsafe-eval'` in prod and document explicitly.
- **Verify**: `curl -I https://<prod>/ | grep -i content-security-policy` shows no `'unsafe-eval'` in prod responses; smoke tests pass.

### C.2 Re-enable `cacheComponents` once stable

- **Addresses**: Not a bug today — opportunity to recover Next.js 16's cache-components benefits.
- **Steps**:
  1. Track Next.js + React Compiler release notes for cache-component fixes.
  2. Once stable, flip `cacheComponents: true` in `next.config.mjs:7`.
  3. Adopt the `use cache` directive on public-viewer RSCs (teacher / class timetable pages) with `cacheTag('class_schedule')` and call `updateTag(...)` from mutating actions in place of `invalidatePublicCache`.
- **Verify**: Public timetables render from cache; admin mutations invalidate within a request cycle.

### C.3 Pick a clear rule: Server Actions vs `/api/*`

- **Addresses**: Architectural inconsistency surfaced during the audit. Today the codebase has both for some mutations.
- **Steps**:
  1. Document the rule: `/api/*` is reserved for **public reads** (timetable JSON for embeds), **webhook/cron endpoints**, and **PDF/Excel export**. Everything else → Server Actions.
  2. Migrate `/api/schedule/auto-arrange`, `/api/schedule/validate-drop` to Server Actions if they're only called from inside the app. Keep the routes that need to be hit from external scripts.
- **Verify**: `bd` issue per migration; manual / E2E coverage after each.

### C.4 Resolve the email-disabled / mailer-wired contradiction

- **Addresses**: M-11.
- **Steps**:
  - Option A — **Disable for real**: Remove `import { sendEmail } from "@/lib/mailer"` from `src/lib/auth.ts`. Replace the outbox-only path with a clearer comment. Strip the disabled UI in `/management/email-outbox`.
  - Option B — **Pick a mailer**: Decide between Resend (modern, generous free tier, native Vercel integration), SES (cheapest at scale), or staying on Azure Communication Email (already wired). Re-enable, update CLAUDE.md to remove the "Email disabled" clause.
- **Verify**: Project policy and code agree.

### C.5 Drop `axios` in favor of native `fetch`

- **Addresses**: L-8.
- **Steps**:
  1. `Grep "from 'axios'"` and rewrite each call site with `fetch` (SWR already wraps `fetch`).
  2. Remove `axios` from `package.json`.
- **Verify**: `pnpm build` size drops; no behavior change.

### C.6 i18n architecture (only if a second language is ever needed)

- Today: Thai hardcoded throughout, Sarabun font loaded for Thai+Latin glyphs.
- If multi-language becomes a requirement, retrofit cost grows quickly. Introduce `next-intl` *before* the requirement lands so messages live in dictionaries from day one.
